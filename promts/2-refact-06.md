Последовательно реализуй план из файла @/.kiro/specs/sonarqube-integration/tasks.md, строго следуя пунктам первого уровня ([ ] 1., [ ] 2., [ ] 3. и т.д.). Перед началом выполнения каждого пункта первого уровня обязательно перечитай документы требований (@/.kiro/specs/sonarqube-integration/requirements.md) и дизайна (@/.kiro/specs/sonarqube-integration/design.md) для сохранения контекста задачи. По мере выполнения обновляй статус задач в файле tasks.md, заменяя [ ] на [x].
При создании структур и функций сначала создавай комментарий в формате go doc с максимально подробным описанием функционала который планируешь реализовать и только потом начинай создавать структуру или реализовывать функцию.
Если в момент начальной реализации функции не весь требуемый функционал можно реализовать добавляй комментарий формата:
// ToDo: необходимо дополнительно реализовать следующий функционал:
// <ссылки на пункты плана, и требований по которым ведется работа>
// <подробное описание функционала который нужно реализовать>
Перед началом работы создай документ impelentation-log.md в который записывай все проблемы которые возникли в процессе реализации плана в формате:
<пункт плана> <Описание возникшей проблемы>
