---- поля    srcServer и dstServer должны получаться из файла dbconfig.yaml по следующему алгоритму:
Например:
нам передано имя базы для восстановления V8_DEV_DSBEKETOV_APK_TOIR3
у нас есть такой файл project.yaml:
``` yaml
# База для для сборки конфигурации в хранилище. Может быть только одна на проект.
store-db: V8_DEV_DSBEKETOV_STORE_ERP
# Список оперативных баз 1С привязанных к проекту.
prod:
# Продуктивная база. Может быть несколько оперативных баз привязанных к одному проекту.
  V8_OPER_APK_TOIR3:
    # Наименование оперативной базы. Будет использоваться в меню выбора базы.
    dbName: "База ТОИР"
    # Заблокированные к установке расширения. Действует для всех привязанных баз
    add-disable:
      - апк_ДоработкиТОИР
    # Привязанные к продуктивной базы тестового контура, демо и дев.
    related:
      V8_DEV_DSBEKETOV_APK_TOIR3:
      V8_DEV_DSBEKETOV_APK_TOIR3_1:

```
и такой файл dbconfig.yaml:
``` yaml
V8_OPER_APK_TOIR3:
    one-server: MSK-AS-010
    prod: true
    dbserver: MSK-SQL-02
V8_DEV_DSBEKETOV_APK_TOIR3:
    one-server: DEV-16-AS-003
    prod: false
    dbserver: MSK-DV-SQL-01
```
из этого следует что:
- srcServer = MSK-SQL-02
- dstServer = MSK-DV-SQL-01

Внеси соответствующие изменения в конфигурационные файл, структуры данных и функции загрузки конфигурации.

## Уточняющие вопросы для точного понимания задачи:

1. **Структура secret.yaml:**
   - Нужно ли полностью удалить секцию `dbrestore` из `secret.yaml` или оставить только пароль?
   Ответ: полностью удалить.
   - Должен ли пароль остаться в `dbrestore.password` или переместиться в `passwords.mssql`?
   Ответ: переместить в `passwords.mssql`.

2. **Перенос полей в app.yaml:**
   - В какую секцию `app.yaml` следует перенести поля `database`, `timeout`, `autotimeout`?
   Ответ: в секцию `dbrestore`.

   - Нужно ли создать новую секцию или добавить в существующую?
   Ответ: добавить секцию `dbrestore`
   - Какие значения по умолчанию установить для этих полей?
   Ответ: 
   - timeout: 50s
   - autotimeout: true


3. **Алгоритм определения srcServer и dstServer:**
   - Как определить продуктивную базу (srcServer) если передано имя тестовой базы?
   Ответ: на основании того что тестовая база находится в секции `related` продуктивной базы.


   - Нужно ли искать в `project.yaml` по всем продуктивным базам и их `related` секциям?
   Ответ: в `project.yaml` тестовая база находится только в одной в секции `related` продуктивной базы 

   - Что делать если база не найдена в `project.yaml` или `dbconfig.yaml`?
   Ответ: вернуть ошибку.

   - Должна ли функция возвращать ошибку или использовать значения по умолчанию?
   Ответ: возвращать ошибку.

4. **Обратная совместимость:**
   - Нужно ли сохранить поддержку старого формата конфигурации?
   Ответ: нет.
   - Как обрабатывать случаи, когда новые поля отсутствуют в конфигурационных файлах?
   Ответ: использовать значения по умолчанию для полей из файла app.yaml, в остальных случаях возвращать ошибку.


5. **Функции загрузки:**
   - Какие именно функции нужно модифицировать: только `LoadDBRestoreConfig` или другие тоже?
   Ответ: проверь все функции загрузки конфигурации.
   - Нужно ли добавить новые функции для поиска связанных баз?
   Ответ: да.
   - Как должна работать приоритизация: централизованная конфигурация → файл конфигурации → переменные окружения?
   Ответ: никак, используй данные из файлов конфигурации.

6. **Тестирование:**
   - Нужно ли обновить существующие тесты?
   Ответ: да.

   - Какие новые тестовые сценарии добавить для проверки алгоритма поиска баз?
   Ответ: 
   - Добавь тест для случая когда база не найдена в `project.yaml` или `dbconfig.yaml`.
   - Добавь тест для случая когда база найдена в `project.yaml` но нет в `dbconfig.yaml`.
   - Добавь тест для случая когда база найдена в `dbconfig.yaml` но нет в `project.yaml`.
   - Добавь тест для случая когда база найдена в `project.yaml` и `dbconfig.yaml`.
