Составь подробный документ **SRS (Software Requirements Specification)** на основе файлов:
- SRS_Document.md
- benadis_runner-requirements-analysis.md

Документ должен описывать функциональные требования новой версии проекта и быть основой для:
- разработки архитектуры системы,
- реализации ПО,
- планирования выполнения проекта.

### Общие правила
1. **Структура документа**:
   - Введение
   - Общее описание системы
   - Функциональные требования
   - Нефункциональные требования
   - Ограничения и условия
   - Требования к интеграциям (1С и внешние системы)
   - Требования к безопасности
   - Глоссарий

2. **Оформление требований**:
   - Использовать шаблоны **EARS** (Ubiquitous, Event-driven, Unwanted Behavior, State-driven, Optional Features, Complex).
   - Для каждого требования:
     - Идентификатор (REQ-XXX)
     - Формулировка по EARS (с ключевыми словами ПРОПИСНЫМИ)
     - Описание
     - Критерии приемки (тесты, метрики, сценарии)
   - Каждое требование должно быть:
     - уникальным,
     - конкретным,
     - тестируемым,
     - однозначным.

3. **Оформление документа**:
   - Использовать заголовки, списки, таблицы.
   - Разделы содержат только требования, относящиеся к их тематике.
   - В конце — **глоссарий** с определением всех терминов.

4. **Специальные требования**:
   - Отдельно указать ограничения: запреты на технологии и методы.
   - В нефункциональных требованиях отразить:
     - производительность,
     - безопасность,
     - надежность.
   - В функциональных требованиях отразить интеграцию:
     - с платформой 1С,
     - с внешними системами.
   - Добавить требования к качеству кода и поддержке:
     - использование лучших практик разработки на Go,
     - читаемость, тестируемость, расширяемость.

---

### Ключевая цель
Сформировать **структурированный и строгий SRS**, который:
- легко использовать для архитектурного проектирования,
- подходит для планирования выполнения,
- понятен LLM (Claude, GPT и др.),
- исключает дублирование и двусмысленность.

По окончании генерации сохрани документ в файл **SRS_for_dev.md**.
