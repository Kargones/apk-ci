Проанализируй функции загрузки конфигурации проекта и разработай архитектурное решение по загружке данных для работы функций приложения в запиши в документ config-arch.md


Решение должно соответствовать следующим требованиям:
- использование 3 файлов конфигурации:
  - config.yaml - файл относящийся к конфигурации приложения. Содержит настройки относящиеся к приложению.

  - project.yaml - файл конфигурации проекта. Содержит настройки относящиеся к проекту. Заготовка для файла конфигурации проекта находится в файле project.yaml.

  - secret.yaml - файл конфигурации секретов. Содержит секреты, которые не должны быть храниться в коде.

Для получения данных о:
- сервере 1С на котором располагается база
- является ли база продуктивной
- сервере MS SQL на котором располагается база
нужно использовать файл dbconfig.yaml. Ключем для поиска является имя базы данных.
