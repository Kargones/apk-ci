Создай функцию которая копирует все каталоги из указанного каталога в другой каталог.

Функция должна принимать следующие аргументы:
- sourceDir - путь к исходному каталогу
- destDir - путь к целевому каталогу в формате UNC `//server.apkholding.ru/share/path`
- username - имя пользователя для аутентификации в AD
- password - пароль пользователя
- domain - домен (apkholding.ru)

Функция должна копировать все каталоги и файлы из sourceDir в destDir, включая все вложенные каталоги без ограничений по глубине. Перед началом копирования выполняется проверка доступности целевого сервера с помощью ping и проверки SMB подключения.

## Технические требования

### Протокол доступа
Целевой каталог расположен на сервере состоящем в домене apkholding.ru. Для доступа к целевому каталогу необходимо использовать протокол SMB/CIFS с аутентификацией в AD apkholding.ru.

### Аутентификация
Использовать Kerberos аутентификацию с автоматическим созданием keytab через kinit с переданными учетными данными для безопасного доступа к домену. Создавать временный файл конфигурации Kerberos (krb5.conf) для операции. Kerberos realm автоматически определяется путем преобразования домена в верхний регистр (APKHOLDING.RU).

### Очистка временных файлов
Временные файлы Kerberos удалять только при успешном завершении операции.

### Обработка ошибок
Реализовать подробное логирование каждой операции с временными метками, используя существующий в проекте логгер в режиме Debug. Логировать каждый скопированный файл и каталог для отслеживания прогресса. Retry механизм: 3 попытки с экспоненциальной задержкой (1, 2, 4 секунды). При сетевых ошибках выполнять немедленное завершение операции. При прерывании операции (Ctrl+C, SIGTERM) выполнять немедленное завершение.

### Обработка специальных файлов
Символические ссылки и специальные файлы (device files, named pipes, sockets) должны пропускаться с логированием предупреждения.

### Сохранение атрибутов файлов
При копировании файлов сохранять время модификации и создания.

### Параметры копирования
Использовать буфер размером 64KB для оптимального сетевого копирования. Файлы размером более 1GB копировать обычным способом без специальной обработки. Для каталогов с большим количеством файлов (>10000) использовать параллельное копирование нескольких файлов.

### Поведение при конфликтах
При обнаружении существующих каталогов в целевом расположении - пропускать их без перезаписи. Существующие файлы также пропускаются без перезаписи.

### Права доступа
Использовать права доступа по умолчанию для скопированных каталогов.

### Ограничения
Установить таймаут операции копирования - 30 минут. Проверка свободного места на целевом сервере не выполняется - полагаться на ошибки системы.

### Технические детали
Приоритетным является использование библиотек Go для реализации функциональности. Функция должна быть создана в пакете `internal/smb` с именем `CopyDirectories`.

### Валидация входных параметров
Выполнять проверку существования исходного каталога и валидность UNC пути перед началом операции копирования.

### Дополнительные зависимости
В случае необходимости установки в контейнер дополнительного ПО создай скрипт установки следующих пакетов:
- samba-client
- krb5-user  
- cifs-utils

Скрипт установки должен быть написан на bash и должен быть выполнен с правами root.

### Возвращаемые значения
Функция должна возвращать структуру с количеством скопированных каталогов и файлов, списком ошибок и временем выполнения операции.

```go
type CopyResult struct {
    CopiedDirectories int
    CopiedFiles      int
    Errors           []error
    Duration         time.Duration
    StartTime        time.Time
    EndTime          time.Time
}
```
