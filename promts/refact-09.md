Проанализируй код проекта и выяви все места в которых из файлов конфигурации загружаются данные в структуры модулей или переменные.
Составь подробный план рефакторинга кода, в результате которого:
- все операции по загрузке из файлов конфигурации будут перенесены в модуль config с сохранением работоспособности существующего функционала
- все операции по чтению данных из конфигурации будут заменены на вызовы функций модуля config
- все операции по запись данных в конфигурацию будут заменены на вызовы функций модуля config
- все операции по чтению данных из переменных окружения будут заменены на вызовы функций модуля config
- все операции по запись данных в переменные окружения будут заменены на вызовы функций модуля config
Реализуй это так чтобы у модуля config не появились дополнительные зависимости от других модулей проекта.
В модуле config должны быть только функции по чтению и запись данных в конфигурацию и переменные окружения.
В модулях проекта, которые используют конфигурацию, должны быть только вызовы функций модуля config.
В модулях проекта, которые используют переменные окружения, должны быть только вызовы функций модуля config.

План запиши в файл tip-refact-09.md
План должен позволять отмечать выполненные пункты.

Если данных для выполнения инструкций недостаточно создай файл quest-refact-09.md и запиши в него все вопросы, после этого останови выполнение инструкций.
